# Lesson 8: 手动验证、打包发布与后续版本路线图

## 本课目标
- 为当前 NoteMark 项目建立可复用的验收方式
- 完成开发、检查、构建、打包的交付流程认知
- 在现有架构上制定清晰的功能演进路线

## 对应 practice 主线
这一课对应 practice 中的项目收尾、手动测试、生产构建和后续增强规划部分。

## 当前仓库映射
- `package.json`
- `resources`
- `out`
- `src/main`
- `src/preload`
- `src/renderer/src`

## 学习重点
- 当前没有自动化测试时，为什么手动 smoke test 必须标准化
- Electron 应用的“能构建”与“能发布”并不是一回事
- 路线规划应该基于当前架构优势，而不是凭空加功能

## 可执行实践路线
1. 固化检查命令
   - `pnpm lint`
   - `pnpm typecheck`
   - `pnpm build`
2. 按真实用户流程做烟雾测试
   - 首次启动是否生成欢迎笔记
   - 新建笔记是否立即可编辑
   - 编辑后是否自动保存
   - 删除后列表和编辑区是否同步更新
   - 关闭重开后数据是否仍在
3. 确认打包链路
   - `pnpm build:win`
   - `pnpm build:mac`
   - `pnpm build:linux`
   - 检查 `resources` 是否完整参与打包
4. 补齐交付文档
   - 功能边界
   - 已知限制
   - 手动验证清单
5. 做版本路线规划
   - 按优先级排功能，不直接堆需求

## 当前版本的建议验收清单
- 启动成功
- 顶栏可拖拽，窗口控制正常
- 列表展示、切换、空状态正常
- 标题和正文可编辑
- 自动保存正常
- 删除后无状态残留
- 重启后数据保留

## 后续功能路线规划

### Phase 1: 核心稳定性
- 保存状态提示
- 删除确认
- 文件名冲突处理
- 异常文件过滤
- 窗口尺寸与位置持久化

### Phase 2: 生产力增强
- 搜索笔记
- 最近编辑排序切换
- 固定笔记
- 快捷键
- Markdown 预览模式

### Phase 3: 笔记管理能力
- 标签系统
- 文件夹或工作区
- 导入导出
- 回收站
- 批量操作

### Phase 4: 高阶扩展
- 全文检索
- 本地索引缓存
- 多窗口
- 插件化编辑器增强
- 同步能力预研

## 本课产出
- 一份可执行的手动验收清单
- 一份构建发布检查表
- 一份按阶段拆分的产品路线图

## 完成标准
- 你知道如何判断当前版本是否可交付
- 你知道接下来该优先做什么，不该急着做什么
- 你能把 NoteMark 的后续迭代分成稳定、提效、管理、扩展四个阶段

## 课程收束建议
完成八课后，建议你回到主线再做一次全流程复盘：
1. 从架构看分层是否清晰
2. 从交互看闭环是否完整
3. 从持久化看边界是否稳定
4. 从路线图看下一版是否聚焦
