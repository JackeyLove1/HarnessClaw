# Lesson 7: 本地 Markdown 存储、CRUD 闭环与启动初始化

## 本课目标
- 在主进程完成真实的笔记文件读写
- 把列表、新建、编辑、删除、重启恢复串成完整闭环
- 明确 NoteMark 当前版本的文件系统策略

## 对应 practice 主线
这一课对应 practice 中的 `fs-extra` 持久化、笔记目录管理、Welcome 笔记初始化和 CRUD 实现部分。

## 当前仓库映射
- `src/main/lib`
- `src/main`
- `src/shared`

## 学习重点
- 为什么本项目把笔记存成 UTF-8 Markdown 文件
- 文件名、标题、内容、更新时间之间如何建立映射
- 主进程中哪些逻辑应该抽到 `lib`，哪些留在 IPC 层

## 目标功能闭环
1. 首次启动时自动创建 `~/NoteMark`
2. 若目录为空，自动生成 `Welcome.md`
3. 读取目录下全部 Markdown 文件
4. 进入应用后可新建笔记
5. 编辑笔记时可保存回文件
6. 删除笔记时同步移除文件
7. 重启应用后内容仍然存在

## 可执行实践路线
1. 设计存储约定
   - 笔记目录位置
   - 支持的文件扩展名
   - 文件命名策略
2. 拆出 main/lib 能力
   - 列表读取
   - 单篇读取
   - 文件写入
   - 新建文件
   - 删除文件
3. 实现初始化逻辑
   - 启动时检查目录
   - 空目录时写入欢迎文档
4. 串回渲染层流程
   - 新建后刷新列表并进入编辑
   - 保存后更新摘要和时间
   - 删除后更新活动笔记
5. 处理边界条件
   - 同名文件冲突
   - 非法文件名
   - 删除最后一篇笔记
   - 空内容笔记的展示策略

## 本课产出
- 一套稳定的文件系统 CRUD 能力
- 一套欢迎文档初始化策略
- 一份笔记目录约定说明

## 完成标准
- `~/NoteMark/*.md` 能稳定作为数据源
- create、edit、autosave、delete、reload 都可验证
- renderer 不需要知道任何路径细节

## 功能规划结果
当前版本坚持“文件优先”而不是“数据库优先”：
- 优点：透明、易备份、易迁移、易调试
- 代价：复杂查询、标签系统、全文检索需要后续设计

## 建议进阶练习
- 为新建笔记加入唯一文件名生成策略
- 为删除操作加入回收站或二次确认
- 为读取列表加入排序和异常文件过滤
