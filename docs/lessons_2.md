# Lesson 2: 桌面窗口、顶栏控制与布局骨架

## 本课目标
- 理解 NoteMark 的桌面应用壳层是如何搭建的
- 完成 frameless 窗口、拖拽顶栏、窗口控制按钮和主布局区域
- 建立“桌面壳层”和“业务内容”分离的实现习惯

## 对应 practice 主线
这一课对应 practice 中的 Electron 窗口初始化、无边框窗口、顶部拖拽栏和基础布局搭建部分。

## 当前仓库映射
- `src/main`：BrowserWindow 配置、窗口行为
- `src/preload/index.ts`：窗口控制方法暴露
- `src/renderer/src/components`：`DraggableTopBar` 等 UI 组件
- `src/renderer/src/assets`：窗口按钮图标和静态资源

## 学习重点
- 为什么 Markdown 工具适合 frameless 窗口设计
- 顶栏拖拽区与可点击控件的冲突如何处理
- 桌面应用布局为什么优先做固定骨架，而不是先堆业务组件

## 可执行实践路线
1. 窗口行为确认
   - 确认窗口是否无边框
   - 确认是否支持拖拽移动
   - 确认关闭、最小化、最大化动作是否经由 preload 暴露
2. 顶栏组件拆分
   - 将顶栏拆成拖拽区域、品牌区域、窗口控制区域
   - 保证按钮区域不被 drag 区覆盖
3. 页面骨架搭建
   - 建立三段式布局：顶栏、左侧笔记栏、右侧编辑区
   - 确保布局高度铺满窗口
   - 确保左右区域在缩放时仍然保持层次稳定
4. 样式基础统一
   - 使用 Tailwind 整理间距、颜色、圆角和边框规则
   - 提取壳层级样式，避免把业务样式写进窗口组件
5. 交互验证
   - 拖动窗口时内容不闪烁
   - 点击控制按钮时事件准确
   - 窗口缩放后布局不塌陷

## 本课产出
- 一个稳定的桌面应用壳层
- 一个职责清晰的 `DraggableTopBar`
- 一份布局约束说明

## 完成标准
- 顶栏支持拖拽
- 按钮点击不受拖拽区域影响
- 主界面具备稳定的左右分栏结构

## 功能规划结果
这一课只解决“容器问题”，不急着扩展业务功能：
- 左侧区域承载笔记列表与创建入口
- 右侧区域承载标题和 Markdown 编辑器
- 顶栏只负责窗口控制，不负责业务逻辑

## 建议进阶练习
- 增加窗口最小宽高限制
- 为活动窗口和非活动窗口设计不同的视觉状态
- 记录布局 tokens，后续给组件统一复用
