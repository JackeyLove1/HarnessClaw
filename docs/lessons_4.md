# Lesson 4: Markdown 编辑器集成与编辑体验设计

## 本课目标
- 在右侧区域建立稳定的 Markdown 编辑体验
- 完成标题输入、正文编辑、内容载入和切换同步
- 理解第三方编辑器在 React 应用里的受控与非受控边界

## 对应 practice 主线
这一课对应 practice 中的编辑器接入、标题编辑、MDXEditor 使用和编辑区域联动部分。

## 当前仓库映射
- `src/renderer/src/components`：编辑器封装组件、标题输入区域
- `src/renderer/src/store`：活动笔记内容、保存动作
- `src/renderer/src/assets`：编辑器图标或样式资源

## 学习重点
- 为什么编辑器封装层要隔离第三方库细节
- 标题和正文为什么通常拆成两个输入域
- 切换笔记时如何避免旧内容残留或光标状态错乱

## 可执行实践路线
1. 集成 Markdown 编辑器
   - 接入 `@mdxeditor/editor`
   - 完成基础主题和正文区域渲染
   - 先保证能读写 Markdown 字符串
2. 标题区与正文区拆分
   - 将标题视为单独字段，而不是正文第一行
   - 标题改动和正文改动都进入统一保存链路
3. 笔记切换时同步内容
   - 切换活动笔记后，编辑器内容应立即更新
   - 确保不会把前一条笔记的内容写回当前笔记
4. 建立编辑反馈
   - 标记当前是否处于编辑中
   - 为保存中、已保存、尚未选择笔记等状态设计反馈
5. 梳理最小编辑能力
   - 标题
   - 段落
   - 列表
   - 引用
   - 代码块

## 本课产出
- 一个可装载 Markdown 内容的编辑器组件
- 一个与标题输入联动的右侧编辑区
- 一份编辑器状态切换说明

## 完成标准
- 选中笔记后，标题和正文都能正确显示
- 修改内容后，渲染层状态能同步更新
- 切换笔记时无内容串写和显示错位

## 功能规划结果
编辑区的第一阶段目标只关注可靠性：
- 稳定加载内容
- 稳定编辑内容
- 稳定切换内容

第二阶段再考虑增强能力：
- 工具栏丰富化
- 快捷键
- 图片粘贴
- Markdown 预览模式

## 建议进阶练习
- 增加字数统计或最近保存时间提示
- 拆分“编辑器容器”和“编辑器适配层”
- 设计编辑器错误回退策略
